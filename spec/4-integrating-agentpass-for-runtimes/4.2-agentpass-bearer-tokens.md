# 4.2. Request a AgentPass Bearer Token

This section defines the Runtime flow for AgentPass Bearer Tokens.

Runtime requests `type=service_bearer_token` and receives an access token for API access.

1. Discover Service Provider support.
2. Call `POST {service_provider_configuration_url}/resolve-deployments` with `request.user.email`.
3. Require one of:
   - `response.authoritative_deployment`, or
   - `response.trusted_federated_deployments`.
4. If `authoritative_deployment` is present, use it.
5. If `trusted_federated_deployments` is present, choose one deployment from that list.
6. POST `{agentpass_configuration_url}/requests` with `type=service_bearer_token` and `user.email` (where `agentpass_configuration_url` is from the selected deployment returned by `POST {service_provider_configuration_url}/resolve-deployments`).
7. Poll until approved.
8. Receive `access_token` and `token_type` (`Bearer`) (optional `cnf`).
9. Call Service Provider with `Authorization: Bearer {access_token}`.

Runtime MUST NOT perform direct authoritative discovery as part of this flow.
