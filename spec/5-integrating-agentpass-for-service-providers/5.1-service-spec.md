# 5.1. AgentPass Service Provider Spec

## Overview

The AgentPass Service Provider Spec defines how a Service Provider validates and enforces delegated authority.

## Token Validation

A Service Provider MUST:

1. Validate token signature
2. Validate issuer trust
3. Validate expiration
4. Enforce scope constraints

If confirmation binding is present, the Service Provider MUST validate proof-of-possession.

## Effective Principal Construction

The Service Provider MUST construct an Effective Principal composed of:

- Delegating Principal
- Agent Principal
- Delegated scope

The Service Provider MUST NOT treat the agent as equivalent to the human unless explicitly granted.

## Audit and Attribution

Service Providers SHOULD:

- Log both human and agent identifiers
- Surface delegated activity distinctly
- Support activity callbacks to AgentPass where feasible

## Authority Resolution

Service Provider MUST provide an authority-resolution endpoint that accepts delegating principal email and returns:

- `trust_mode = "authoritative"` and resolved authoritative authority when authoritative discovery succeeds, or
- `trust_mode = "federated"` and trusted federated authority options when authoritative discovery is unavailable.

If authoritative discovery succeeds, Service Provider MUST NOT return federated options.

If an Authoritative AgentPass is configured for the Delegating Principal, Service Provider MUST only authorize agents delegated by that Authoritative AgentPass.
