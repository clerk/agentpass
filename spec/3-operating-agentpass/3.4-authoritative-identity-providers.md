# 3.4. Authoritative Identity Providers

Authoritative Identity Providers are discovered via DNS on `delegating_principal_domain`, typically by Service Provider during deployment resolution and precedence enforcement.

If an authoritative AgentPass deployment is configured for the Delegating Principal, Service Provider MUST treat it as authoritative and MUST only authorize agents delegated by that authoritative deployment, as defined in:

- [`3.1-authoritative-vs-federated.md`](3.1-authoritative-vs-federated.md)

Implementations MUST:
- publish `_agentpass-authoritative.{domain}` TXT with URL value
- serve `GET {e}` metadata
- support required AgentPass endpoints (see `3.6-endpoints.md`)
- enforce policy and approvals
- issue artifacts/tokens containing `delegating_principal_domain`
