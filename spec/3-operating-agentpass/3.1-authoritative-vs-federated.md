# 3.1. Authoritative vs Federated

Each AgentPass deployment operates in one of two trust modes:

- `authoritative`
- `federated`

An AgentPass deployment MUST declare its `trust_mode` in its metadata (`GET {e}`) and MUST publish a DNS discovery record consistent with that mode (Section `3.2`).

## 3.1.1. Authoritative Trust Mode

`trust_mode = "authoritative"` (an **authoritative AgentPass deployment**) is intended for organizations that need enterprise control over how their members delegate to agents, including policy enforcement, approval control, and auditable attribution.

In authoritative mode, the AgentPass deployment discovered for `delegating_principal_domain` via `_agentpass-authoritative.{domain}` is the source of truth for delegation policy and authorization for that domain.

For interoperable implementations, authoritative mode is bootstrapped via DNS TXT discovery (Section `3.2`) followed by metadata fetch (`GET {e}`).

## 3.1.2. Federated Trust Mode

`trust_mode = "federated"` (a **federated AgentPass deployment**) is used when the Service Provider does not have a usable authoritative deployment for the Delegating Principal and elects to trust one or more federated deployments by policy.

Federated providers are high-trust dependencies and should be explicitly selected by Service Provider policy.

## 3.1.3. Precedence Rule (Normative)

The precedence rule is enforced by the Service Provider when accepting artifacts/tokens.

If authoritative DNS TXT discovery succeeds for `delegating_principal_domain`, the Service Provider MUST attempt to use the discovered authoritative deployment for that Delegating Principal.

If anything after DNS TXT discovery fails (for example, HTTPS metadata fetch or validation), the Service Provider MAY fall back to a federated trust mode for that Delegating Principal.

When the Service Provider operates in authoritative mode for a Delegating Principal, it MUST accept only artifacts/tokens whose `iss` matches the authoritative deployment `issuer` obtained from metadata for the discovered endpoint, and whose signatures validate using the authoritative deployment's published keys.

When the Service Provider operates in federated mode, it MUST accept only artifacts/tokens issued by federated deployments it explicitly selected by policy (for example, deployments returned as trusted options for that Delegating Principal by the Service Provider deployment-resolution endpoint).
