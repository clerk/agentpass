# 3.4. Authoritative Identity Providers

Authoritative Identity Providers are discovered via DNS on `delegating_principal_domain`, typically by Service Provider during authority resolution and precedence enforcement.

If an Authoritative AgentPass is configured for the Delegating Principal, Service Provider MUST operate in `trust_mode = "authoritative"` and MUST only authorize agents delegated by that Authoritative AgentPass, as defined in:

- [`3.1-authoritative-vs-federated.md`](3.1-authoritative-vs-federated.md)

Implementations MUST:
- publish `_agentpass-authoritative.{domain}` TXT with URL value
- serve `GET {e}` metadata
- support required AgentPass endpoints (see `3.6-endpoints.md`)
- enforce policy and approvals
- issue artifacts/tokens containing `delegating_principal_domain`
