# 5.3. Support AgentPass Bearer Tokens

Service Providers validate bearer tokens issued by AgentPass.

For bearer-token flows, Service Providers MUST expose and enforce the same deployment-resolution behavior defined for browser sessions (`POST {e}/resolve-deployments`).

If `trust_mode = "federated"` for the delegating principal domain, Service Provider MUST validate that issuer is in explicit trusted federated allowlist policy.

If `trust_mode = "authoritative"` for the delegating principal domain, Service Provider MUST reject federated issuers.

If an authoritative AgentPass deployment is configured for the Delegating Principal, Service Provider MUST only authorize agents delegated by that authoritative deployment.

Service Providers MUST validate:
- signature (JWKS)
- trusted `iss` (and enforce authoritative precedence)
- `aud` matches the Service Provider
- `exp`
- scope/capabilities
- proof-of-possession if `cnf` is present

Service Providers MUST log delegating principal + agent principal.
