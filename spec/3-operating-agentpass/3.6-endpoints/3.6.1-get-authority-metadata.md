# 3.6.1. GET {e} (Deployment Metadata)

Returns the AgentPass deployment metadata document.

Schema: `../3-schemas/authority-metadata.schema.json`.

## Request

Client issues `GET {e}` resolved from discovery.

Client SHOULD send `Accept: application/json`.

## Response

Deployment MUST return JSON metadata with:

- `kind = "deployment"`
- `issuer`
- `trust_mode`
- `endpoints.requests`
- `endpoints.request_status`

Optional fields include:

- `endpoints.events` for SSE support
- `crypto.jwks_uri` and supported algorithms
- approval policy hints

## Endpoint Semantics

`endpoints.requests`:

- absolute HTTPS URL for `POST` request creation

`endpoints.request_status`:

- absolute HTTPS URL template for request status lookup
- template variable `{id}` identifies request id

`endpoints.events` (optional):

- absolute HTTPS URL template for SSE status events
- template variable `{id}` identifies request id

## Caching and Freshness

Clients SHOULD cache metadata for a bounded duration.

Clients MUST re-fetch metadata when protocol-critical validation fails, issuer configuration changes, or endpoint references become invalid.

## Errors

Suggested status classes:

- `404` endpoint not found
- `406` unsupported `Accept`
- `5xx` transient deployment failure
