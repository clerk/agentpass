# 5.6.3. Artifact Redemption Endpoint

Redeems an AgentPass Browser Sessions artifact and returns a single-use `initialization_url`.

Schema: `../../3-operating-agentpass/3-schemas/service-init.schema.json`.

Endpoint URL is provided by Service Provider configuration:

- `endpoints.init` -> `POST {service_provider_artifact_redemption_url}`

## Request

Service Provider MUST accept JSON body conforming to `service-init` request schema.

Required fields:

- `request.artifact.type`
- `request.artifact.value`

Service Provider SHOULD accept:

- `request.runtime_proof`
- `request.user.email`

## Validation Requirements

Before issuing `initialization_url`, Service Provider MUST validate:

- artifact integrity and freshness
- single-use semantics (artifact not previously consumed)
- audience matches this Service Provider
- issuer trust policy
- authoritative precedence for User email domain (derived from `request.user.email`)
- runtime proof, when required

Normative validation behavior is defined in:

- [`../5.4-agentpass-browser-sessions.md`](../5.4-agentpass-browser-sessions.md)

## Response

On success, Service Provider MUST return JSON conforming to `service-init` response schema, including:

- `response.initialization_url`

## Errors

Suggested status classes:

- `400` malformed request
- `401` runtime proof failure
- `403` trust/policy rejection
- `409` replay/single-use conflict
- `422` invalid artifact
