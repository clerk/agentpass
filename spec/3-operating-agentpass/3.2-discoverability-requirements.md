# 3.2. Discoverability requirements

AgentPass uses DNS TXT records to bootstrap discovery of HTTPS endpoints.

Given `delegating_principal_domain` (for example, `example.com`):

- For an authoritative AgentPass deployment (`trust_mode = "authoritative"`), TXT record MUST be set at `_agentpass-authoritative.example.com`.
- For a federated AgentPass deployment (`trust_mode = "federated"`), TXT record MUST be set at `_agentpass-federated.example.com`.

The TXT value MUST be a URL and MUST NOT use key/value formatting.

Example value:

```txt
https://agentpass.example.com/ap
```

The discovered URL is requested for metadata using `GET {e}`.

## Parsing

Clients MUST reject records that:

- are not an `https://` URL

If multiple TXT values exist for a lookup name, clients SHOULD fail unless exactly one value parses successfully.

## Security Notes

DNS discovery is a bootstrap mechanism and should not be treated as strong identity proof in the absence of DNSSEC.

Clients SHOULD validate DNSSEC when available.
