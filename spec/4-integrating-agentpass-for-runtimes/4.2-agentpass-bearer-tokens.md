# 4.2. Request a AgentPass Bearer Token

This section defines the Runtime flow for AgentPass Bearer Tokens.

Runtime requests `type=service_bearer_token` and receives an access token for API access.

1. Discover Service Provider support.
2. Call `POST {service_e}/resolve-authority` with `request.delegating_principal_email`.
3. Require `response.trust_mode`.
4. If `trust_mode=authoritative`, treat the Delegating Principal as authoritatively configured and use returned authoritative authority.
5. If `trust_mode=federated`, choose one issuer from `trusted_federated_authorities`.
6. Validate `resolution`/`trust_mode` consistency (`authoritative` <-> `authoritative`, `federated_options` <-> `federated`).
7. POST `{authority_e}/requests` with `type=service_bearer_token` and returned `delegating_principal_domain`.
8. Poll until approved.
9. Receive `access_token` and `token_type` (`Bearer`) (optional `cnf`).
10. Call Service Provider with `Authorization: Bearer {access_token}`.

Runtime MUST NOT perform direct authoritative discovery as part of this flow.
