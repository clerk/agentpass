# 2.1. Definitions

This section defines terms used across the AgentPass specification.

## Core Terms

### User
An identity representing the account at a Service Provider that an agent is delegated to operate on behalf of.

The User is the subject of delegation. Approval MAY be granted by the User or by another approver (for example, the User's manager or an automated policy system).

### Agent
A software identity that acts on behalf of a User.

### Runtime
The execution environment that requests artifacts/tokens from AgentPass and presents them to Service Providers.

### Service Provider
The relying party application that accepts AgentPass artifacts or tokens.

### AgentPass deployment
One of the many deployed AgentPass instances on the internet. For a given User, the acceptable AgentPass deployment(s) are determined by the Service Provider (via discovery and its deployment-resolution logic), and the selected deployment governs delegation policy and issues artifacts/tokens.

### Authoritative AgentPass
An AgentPass deployment with `trust_mode = "authoritative"`. An Authoritative AgentPass is intended to be the source of truth for delegation policy for one or more User email domains and is discoverable via the `_agentpass.{user_email_domain}` DNS record, where `user_email_domain` is derived from the User email address.

### Federated AgentPass
An AgentPass deployment with `trust_mode = "federated"`. A Federated AgentPass is not discoverable via User DNS and is used only when explicitly trusted by Service Provider policy (for example, when returned as a trusted option from `POST {service_provider_configuration_url}/resolve-deployments`).

### `user.email`
The User email address the Runtime intends to sign in as. Runtime sends this to the Service Provider deployment-resolution endpoint.

### Approver
The entity that ultimately approves or denies a delegation request. An approver MAY be the User, another human (for example, a manager), or an automated policy system.

### `trust_mode`
The trust mode of an AgentPass deployment, declared in AgentPass configuration (`GET {agentpass_configuration_url}`). Valid values are:
- `authoritative`: the deployment is authoritative for one or more User email domains and is intended to be discovered via `_agentpass.{user_email_domain}`, where `user_email_domain` is derived from `user.email`.
- `federated`: the deployment is not authoritative for User email domains and is intended to be used only when explicitly trusted by Service Provider policy.

### Artifact
A short-lived delegated credential intended for a specific redemption flow (for example, AgentPass Browser Sessions via `POST {service_provider_configuration_url}/init`).

### Bearer Token
A delegated access token presented to Service Provider APIs using `Authorization: Bearer`.

### AgentPass Configuration URL
The HTTPS URL at which `GET` returns AgentPass configuration for an AgentPass deployment.

In URI templates, this is written as `{agentpass_configuration_url}`.

### Service Provider Configuration URL
The HTTPS URL at which `GET` returns Service Provider configuration for a Service Provider.

In URI templates, this is written as `{service_provider_configuration_url}`.

### AgentPass Configuration
The JSON configuration document returned by `GET {agentpass_configuration_url}` that describes an AgentPass deployment's issuer identity, `trust_mode`, endpoints, and cryptographic material.

### Service Provider Configuration
The JSON configuration document returned by `GET {service_provider_configuration_url}` that describes Service Provider trust configuration and AgentPass integration endpoints.

### Audience (`aud`)
The intended Service Provider recipient for an artifact/token. Service Providers MUST reject artifacts/tokens not addressed to their own origin/identifier.

### Scope
The abstract delegated permission boundary approved for agent execution.

### Capabilities
The Service Provider-specific expression of delegated scope (for example, operation names or permissions). Capabilities MUST be enforced by Service Providers at authorization time.

### Effective Principal
The combined execution identity formed by User + Agent + approved delegated scope/capabilities.

### `issuer` (`iss`)
The identity of the issuing authority for an artifact/token. Service Providers validate issuer trust and precedence.

### `cnf` (Confirmation)
A confirmation claim used to bind an artifact/token to a holder key or runtime key material.

### Proof-of-Possession (PoP)
A runtime/Service Provider verification step proving the caller controls the key material referenced by `cnf`.

### SSE (Server-Sent Events)
A one-way streaming mechanism over HTTP used by AgentPass implementations for optional request status push updates.

### `initialization_url`
A short-lived, single-use URL returned by a Service Provider after artifact redemption and used to establish an agent-attributed browser session.

### Single-use
A credential or URL property indicating one-time redeemability. Subsequent redemption attempts MUST be rejected.
