# 5.4.5. POST {e}/resolve-deployments

Resolves the acceptable AgentPass deployment(s) for a delegating principal email address.

Schema: `../../3-operating-agentpass/3-schemas/service-authority-resolution.schema.json`.

Examples:

- `../../3-operating-agentpass/3-examples/service-authority-resolution-authoritative.json`
- `../../3-operating-agentpass/3-examples/service-authority-resolution-federated-options.json`

## Request

Service Provider MUST accept JSON body containing:

- `request.delegating_principal_email`

## Service Provider Behavior

Service Provider MUST:

1. Derive `delegating_principal_domain` from the email domain.
2. Determine whether an authoritative AgentPass deployment is configured for that Delegating Principal by performing authoritative discovery for that domain (`_agentpass-authoritative.{domain}`).
3. If authoritative discovery succeeds, return that authoritative deployment.
4. If authoritative discovery does not resolve, discover federated fallback for that domain (`_agentpass-federated.{domain}`) and return only trusted federated deployment options.

Service Provider MUST NOT return federated options when authoritative discovery resolves successfully.
If an authoritative AgentPass deployment is configured for the Delegating Principal, Service Provider MUST only authorize agents delegated by that authoritative deployment.

## Response

Response includes:

- `response.delegating_principal_domain`

If authoritative discovery succeeds:

- `response.authoritative_deployment` MUST be present.

If authoritative discovery does not resolve:

- `response.trusted_federated_deployments` MUST be present and non-empty.
- each returned federated deployment MUST be explicitly selected by Service Provider policy.

Service Provider MUST treat federated providers as high-trust dependencies and MUST NOT return implicit federated options.

## Errors

Suggested status classes:

- `400` malformed request or invalid email
- `404` no authoritative provider and no trusted federated options
- `422` semantically invalid resolution request
- `5xx` transient Service Provider failure
