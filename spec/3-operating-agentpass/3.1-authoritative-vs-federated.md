# 3.1. Authoritative vs Federated

AgentPass operates in one of two trust modes:

- `authoritative`
- `federated`

## 3.1.1. Authoritative Trust Mode

`trust_mode = "authoritative"` (an **Authoritative AgentPass**) is intended for organizations that need enterprise control over how their members delegate to agents, including policy enforcement, approval control, and auditable attribution.

In authoritative mode, the Delegating Principal's domain-resolved AgentPass is the source of truth for delegation policy and authorization.

For interoperable implementations, authoritative configuration is established by authoritative discovery for `delegating_principal_domain`.

## 3.1.2. Federated Trust Mode

`trust_mode = "federated"` (a **Federated AgentPass**) is used when no Authoritative AgentPass is resolved for the Delegating Principal and a Service Provider elects to trust a federated provider by policy.

Federated providers are high-trust dependencies and should be explicitly selected by Service Provider policy.

## 3.1.3. Precedence Rule (Normative)

If an Authoritative AgentPass is discoverable for the Delegating Principal, the Service Provider MUST only accept agents authorized by the Authoritative AgentPass.

A Federated AgentPass MAY be used as a fallback only when an Authoritative AgentPass is not discoverable. The Service Provider MUST explicitly define which Federated AgentPass deployments are acceptable as a fallback within its configuration.
