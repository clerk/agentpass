# 5.4.5. POST {e}/resolve-authority

Resolves the AgentPass authority for a delegating principal email address.

Schema: `../../3-operating-agentpass/3-schemas/service-authority-resolution.schema.json`.

Examples:

- `../../3-operating-agentpass/3-examples/service-authority-resolution-authoritative.json`
- `../../3-operating-agentpass/3-examples/service-authority-resolution-federated-options.json`

## Request

Service Provider MUST accept JSON body containing:

- `request.delegating_principal_email`

## Service Provider Behavior

Service Provider MUST:

1. Derive `delegating_principal_domain` from the email domain.
2. Determine whether an Authoritative AgentPass is configured for that Delegating Principal by performing authoritative discovery for that domain (`_agentpass-authoritative.{domain}`).
3. If authoritative discovery succeeds, return that authoritative authority.
4. If authoritative discovery does not resolve, discover federated fallback for that domain (`_agentpass-federated.{domain}`) and return only trusted federated authority options.

Service Provider MUST NOT return federated options when authoritative discovery resolves successfully.
If an Authoritative AgentPass is configured for the Delegating Principal, Service Provider MUST only authorize agents delegated by that Authoritative AgentPass.

## Response

Response includes:

- `response.delegating_principal_domain`
- `response.trust_mode`
- `response.resolution`

If `response.trust_mode = "authoritative"`:

- `response.resolution` MUST be `"authoritative"`.
- `response.authoritative_authority` MUST be present.

If `response.trust_mode = "federated"`:

- `response.resolution` MUST be `"federated_options"`.
- `response.trusted_federated_authorities` MUST be present and non-empty.
- each returned federated authority MUST be explicitly allowlisted by Service Provider policy.

Service Provider MUST treat federated providers as high-trust dependencies and MUST NOT return implicit federated options.

## Errors

Suggested status classes:

- `400` malformed request or invalid email
- `404` no authoritative provider and no trusted federated options
- `422` semantically invalid resolution request
- `5xx` transient Service Provider failure
