# 2.1. Definitions

This section defines terms used across the AgentPass specification.

## Core Terms

### Delegating Principal
A human identity that delegates authority to an agent.

### Agent Principal
A software identity that acts on behalf of a delegating principal.

### Runtime
The execution environment that requests artifacts/tokens from AgentPass and presents them to Service Providers.

### Service Provider
The relying party application that accepts AgentPass artifacts or tokens.

### AgentPass Authority
One of the many deployed AgentPass instances on the internet. For a given delegating principal, the selected AgentPass Authority is determined by Service Provider authority resolution and/or authoritative discovery, and governs delegation policy and issues artifacts/tokens.

### `delegating_principal_domain`
The DNS domain associated with the delegating principal. Service Provider derives this from delegating principal email for authority resolution and uses it for authoritative discovery and federated precedence enforcement.

### `delegating_principal_email`
The user email the Runtime intends to sign in as. Runtime sends this to Service Provider authority-resolution endpoint.

### Authoritative Identity Provider
The AgentPass authority discovered for `delegating_principal_domain` via DNS by Service Provider.

### Federated Identity Provider
A non-authoritative provider that a Service Provider may trust by local policy, subject to authoritative precedence rules.

### `trust_mode`
The trust mode applied by a Service Provider for a Delegating Principal after authority resolution. Valid values are:
- `authoritative`: an Authoritative AgentPass is configured; the authoritative provider MUST be used.
- `federated`: no Authoritative AgentPass is configured; Service Provider MAY allow only explicitly trusted federated providers.

### Artifact
A short-lived delegated credential intended for a specific redemption flow (for example, AgentPass Browser Sessions via `POST {service_e}/init`).

### Bearer Token
A delegated access token presented to Service Provider APIs using `Authorization: Bearer`.

### Authority Metadata
The JSON metadata document returned by `GET {authority_e}` that describes an AgentPass authority's issuer identity, endpoints, and cryptographic material.

### Service Provider Metadata
The JSON metadata document returned by `GET {service_e}` that describes Service Provider trust configuration and AgentPass integration endpoints.

### Audience (`aud`)
The intended Service Provider recipient for an artifact/token. Service Providers MUST reject artifacts/tokens not addressed to their own origin/identifier.

### Scope
The abstract delegated permission boundary approved for agent execution.

### Capabilities
The Service Provider-specific expression of delegated scope (for example, operation names or permissions). Capabilities MUST be enforced by Service Providers at authorization time.

### Effective Principal
The combined execution identity formed by delegating principal + agent principal + approved delegated scope/capabilities.

### `issuer` (`iss`)
The identity of the issuing authority for an artifact/token. Service Providers validate issuer trust and precedence.

### `cnf` (Confirmation)
A confirmation claim used to bind an artifact/token to a holder key or runtime key material.

### Proof-of-Possession (PoP)
A runtime/Service Provider verification step proving the caller controls the key material referenced by `cnf`.

### SSE (Server-Sent Events)
A one-way streaming mechanism over HTTP used by AgentPass implementations for optional request status push updates.

### `initialization_url`
A short-lived, single-use URL returned by a Service Provider after artifact redemption and used to establish an agent-attributed browser session.

### Single-use
A credential or URL property indicating one-time redeemability. Subsequent redemption attempts MUST be rejected.
